MODULE 4

Status Codes & Behavior

Status codes are not just success/failure signals. They often leak valuable information during testing.

Learning Objectives

  • Understand what different status code classes mean in practice
  • Learn how to use status codes as oracles during testing
  • Identify information leakage through error responses

Key Status Code Classes

2xx - Success

200, 201, 204 are common. Inconsistent behavior between users can reveal access control issues.

3xx - Redirection

Open redirect vulnerabilities often live here. Also check if sensitive headers are sent to third parties.

4xx - Client Errors

400, 401, 403, 404. Differences in responses between valid/invalid users can lead to user enumeration or IDOR confirmation.

5xx - Server Errors

Stack traces, internal paths, and framework versions are frequently leaked here. Goldmine for reconnaissance.

Practical Testing Tips

  • Compare responses for the same request as different users.
  • Send malformed input and observe error messages.
  • Look for timing differences that can act as oracles.
  • Never ignore 5xx responses during testing.
