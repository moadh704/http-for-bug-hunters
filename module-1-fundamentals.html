MODULE 1

HTTP Fundamentals

Before you can hunt effectively, you need to understand how HTTP actually works under the hood.

Learning Objectives

  • Understand the request-response cycle
  • Know the structure of HTTP requests and responses
  • Recognize what parts of HTTP are attacker-controlled
  • Build mental models for testing at the protocol level

The Shape of an HTTP Transaction

Every interaction on the web follows the same basic pattern:

  1. Client builds a request
  2. Server receives, parses, and authorizes it
  3. Server builds and sends a response
  4. Client receives and processes the response

Each of these four steps is a potential point of failure or exploitation. Most web vulnerabilities exist because one of these steps makes incorrect assumptions.

Anatomy of a Request

An HTTP request consists of four parts:

GET /api/users/123 HTTP/1.1
Host: example.com
Authorization: Bearer eyJhbGci...
Content-Type: application/json

{"action": "view"}

Key Parts:

  • Request Line: Method + Path + HTTP Version
  • Headers: Metadata about the request
  • Blank Line: Separator
  • Body (optional): Data being sent

Security Note: Almost everything in a request is attacker-controlled input until the server validates it. This is the foundation of most web attacks.

Anatomy of a Response

HTTP/1.1 200 OK
Content-Type: application/json
Set-Cookie: session=abc123; HttpOnly; Secure

{"id": 123, "name": "John"}

Responses often leak more information than developers intend (stack traces, versions, internal paths, etc.). Learning to read responses carefully is a core skill.
