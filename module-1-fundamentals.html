HTTP Fundamentals

HTTP is the foundation of the web. If you don’t deeply understand how it works, you will miss many vulnerabilities that exist purely at the protocol level.

Learning Objectives

  • Understand the full request-response lifecycle
  • Know exactly what is attacker-controlled in an HTTP request
  • Learn how to read raw HTTP messages effectively
  • Build a mental model for finding issues at the protocol layer

The Request-Response Cycle

Every web interaction follows a simple but powerful pattern:

1. Client builds the request
2. Server receives and processes it
3. Server builds and sends the response
4. Client receives and renders the response

Most vulnerabilities happen because one of these steps makes an incorrect assumption about the data it receives or sends. Your job as a hunter is to break those assumptions.

Anatomy of an HTTP Request

A request has four parts:

GET /api/users/123 HTTP/1.1
Host: api.example.com
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
User-Agent: Mozilla/5.0...
Content-Type: application/json

{"include": "sensitive"}

Breakdown:

  • Request Line: Method, path, and HTTP version
  • Headers: Key-value metadata (many are attacker-controlled)
  • Empty line: Required separator
  • Body: Optional data (JSON, form data, XML, etc.)

Important for hunters: Almost every part of the request can be modified. The server must validate everything.

Anatomy of an HTTP Response

HTTP/1.1 200 OK
Content-Type: application/json
Set-Cookie: session=abc123; HttpOnly; Secure; SameSite=Lax
X-Content-Type-Options: nosniff

{"user": {"id": 123, "role": "admin"}}

Responses can leak sensitive information through headers, status codes, or the body itself. Always analyze the full response during testing.

Key Security Concepts

  • Everything is input until validated — Treat every header, parameter, and cookie as potentially malicious.
  • Responses leak information — Status codes, headers, and error messages often reveal internal details.
  • Trust boundaries — The point where the server stops trusting client input is where many bugs live.

Key Takeaways

  • HTTP is simple but extremely powerful for attackers when assumptions break.
  • Master reading raw requests and responses — this skill will help you in almost every engagement.
  • Never assume the server properly validates what it receives.
