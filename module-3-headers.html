Headers Mastery

Headers are one of the richest areas for finding security issues. Many vulnerabilities hide in how headers are handled or ignored.

Learning Objectives

  • Understand key request and response headers
  • Know which headers have security implications
  • Learn how to test header-related vulnerabilities
  • Identify misconfigurations in security headers

Important Request Headers

  • Authorization — Tokens, API keys. Look for weak tokens, missing expiry, or leakage.
  • Cookie — Session management. Check Secure, HttpOnly, SameSite flags on responses.
  • Content-Type — Can sometimes be manipulated to reach different code paths.
  • X-Forwarded-For / X-Real-IP — Often spoofable. Dangerous if used for access control.
  • Origin / Referer — Important for CORS and CSRF testing.

Important Response Headers

  • Set-Cookie — Session security flags.
  • Content-Security-Policy (CSP) — Critical for XSS prevention.
  • Strict-Transport-Security (HSTS) — Prevents downgrade attacks.
  • X-Content-Type-Options — Should be nosniff.
  • Access-Control-Allow-Origin — CORS misconfigurations are very common.

Key Takeaways

  • Headers are often overlooked during testing.
  • Many access control issues come from trusting client-controlled headers.
  • Security headers are frequently missing or misconfigured.
  • Always check how custom headers are handled by the application.

Practice Recommendations

Focus on CORS, CSP, and cookie security labs on PortSwigger.
