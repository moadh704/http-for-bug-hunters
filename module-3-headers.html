MODULE 3

Headers Mastery

HTTP headers carry critical metadata. Many serious vulnerabilities live in how headers are processed or ignored.

Learning Objectives

  • Understand the security implications of key request and response headers
  • Learn how to test for header-related vulnerabilities
  • Identify common misconfigurations in security headers

Important Request Headers

Authorization

Carries tokens and API keys. Look for weak tokens, missing expiry, or leakage.

X-Forwarded-For / X-Real-IP

Often spoofable. Dangerous if used for access control or rate limiting.

Content-Type

Can sometimes be manipulated to reach different (less validated) code paths.

Important Response Headers

Set-Cookie

Controls session security. Missing Secure, HttpOnly, or SameSite flags are common findings.

Content-Security-Policy (CSP)

Strong defense against XSS. Look for overly permissive policies.

Access-Control-Allow-Origin

CORS misconfigurations are very common and often high severity.
