Headers Mastery

HTTP headers carry critical metadata. Many serious vulnerabilities (CORS misconfigurations, cache poisoning, authentication bypasses, XSS, etc.) live in how headers are processed or ignored.

Learning Objectives

  • Understand the security implications of key request and response headers
  • Learn how to test for header-related vulnerabilities
  • Identify common misconfigurations in security headers
  • Recognize when applications trust client-controlled headers unsafely

Important Request Headers

Authorization

Carries authentication credentials (Bearer tokens, API keys, Basic auth, etc.).

Testing Focus: Weak or predictable tokens, missing token expiry, token leakage in logs or Referer headers, algorithm confusion in JWTs.

Cookie

Used for session management. The real security is usually enforced on the corresponding Set-Cookie response header.

Testing Focus: Missing Secure, HttpOnly, or SameSite flags.

Content-Type

Tells the server how to parse the body. Changing this can sometimes reach different (less validated) code paths.

Testing Focus: Content-Type confusion attacks, reaching XML/JSON parsers unexpectedly.

X-Forwarded-For / X-Real-IP / X-Forwarded-Host

Commonly used behind proxies. These headers are trivial to spoof.

Testing Focus: IP-based access control bypass, password reset poisoning, rate limit bypass.

Origin & Referer

Important for CORS and CSRF protection mechanisms.

Testing Focus: CORS misconfigurations, Referer-based CSRF protection bypass.

Important Response Headers

Set-Cookie

Controls session security. Missing flags are very common findings.

Content-Security-Policy (CSP)

The strongest client-side defense against XSS. Look for overly permissive policies (unsafe-inline, unsafe-eval, wildcards).

Strict-Transport-Security (HSTS)

Forces HTTPS. Missing or weak HSTS allows downgrade attacks.

Access-Control-Allow-Origin

Critical for CORS. Reflecting arbitrary origins while allowing credentials is a high-severity issue.

X-Content-Type-Options

Should be set to nosniff. Prevents MIME type sniffing attacks.

Key Takeaways

  • Headers are frequently overlooked but extremely high-value for testing.
  • Many access control issues stem from blindly trusting client-controlled headers.
  • Security headers (CSP, HSTS, etc.) are often missing or misconfigured in real applications.
  • Always check how custom or non-standard headers are handled.
