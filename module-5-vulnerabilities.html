Finding Vulnerabilities at the HTTP Layer

Many vulnerabilities can be discovered simply by carefully observing and manipulating HTTP traffic.

Learning Objectives

  • Map common vulnerability categories to HTTP-level behavior
  • Develop a practical methodology for testing at the protocol level

Common Vulnerability Patterns

Broken Access Control

The most common category in bug bounties. Involves manipulating IDs, roles, or tokens.

Injection

SQL, command, NoSQL, template injection through user-controlled parameters.

Business Logic Flaws

Workflow bypasses, price manipulation, race conditions — often found by deviating from expected request sequences.
