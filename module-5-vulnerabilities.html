MODULE 5

Finding Vulnerabilities at the HTTP Layer

Many vulnerabilities can be discovered simply by carefully observing and manipulating HTTP traffic.

Learning Objectives

  • Map common vulnerability types to HTTP behavior
  • Develop a methodology for testing at the protocol level
  • Understand how OWASP Top 10 categories manifest in HTTP

Common Vulnerability Patterns

Broken Access Control

Manipulating IDs, roles, or tokens in requests. The most common category in bug bounties.

Injection

SQL, command, NoSQL, template injection through user-controlled parameters in requests.

Security Misconfiguration

Missing security headers, verbose error messages, exposed sensitive endpoints.

Business Logic Flaws

Workflow bypasses, price manipulation, race conditions — often found by deviating from expected request sequences.

Testing Mindset

Good hunters treat every request as potentially attacker-controlled and every response as potentially leaky.

Ask yourself constantly: "What would happen if I changed this?"
