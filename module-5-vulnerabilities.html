MODULE 5

Finding Vulnerabilities at the HTTP Layer

Many vulnerabilities can be discovered simply by carefully observing and manipulating HTTP traffic. This module focuses on patterns that appear frequently in real bug bounty programs and pentests.

Learning Objectives

  • Map common vulnerability categories to HTTP-level behavior
  • Develop a practical methodology for testing at the protocol level
  • Understand how OWASP Top 10 issues manifest in HTTP requests and responses

Common Vulnerability Patterns at HTTP Level

Broken Access Control

The #1 category in bug bounties. Usually involves manipulating IDs, roles, or tokens in requests.

Testing Approach: Change object IDs, test horizontal and vertical privilege escalation, check for BOLA/BFLA issues.

Injection

SQL, command, NoSQL, template, LDAP injection through user-controlled parameters.

Testing Approach: Supply special characters and observe behavior changes or error messages.

Security Misconfiguration

Missing security headers, verbose error messages, exposed sensitive endpoints or files.

Business Logic Flaws

Workflow bypasses, price/rate manipulation, race conditions. These often require deviating from the expected request sequence.

Testing Mindset

Good hunters treat every request as potentially attacker-controlled and every response as potentially leaky. Constantly ask: "What if I change this?"

Practice Recommendations

Work through PortSwigger labs on access control, injection, and business logic.
